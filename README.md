# Error Analytics - 로그 에러 분석 도구

서버 로그 파일에서 작업(Job) 성공/실패 현황과 에러코드를 자동으로 집계하여 CSV 리포트를 생성하는 Python 스크립트입니다.

## 주요 기능

- 다중 서버 로그 파일 일괄 분석
- 에러코드별 발생 건수 집계 및 `Caused by` 원인 추적
- TSA - GPKI(GpkiLocalAmanoService) 서명/검증 에러 별도 집계
- **서버별 CSV 리포트** 생성
- **날짜별 통합 CSV 리포트** 생성 (전체 서버 합산)

## 디렉토리 구조

```
error_analytics/
├── analyze_errors.py    # 메인 분석 스크립트
├── error_codes.py       # 에러코드 → 설명 매핑 사전
├── pg/                  # 로그 파일 디렉토리 (기본 경로)
│   ├── 1/               # 서버 1
│   │   ├── pg.log.2026-04-01.0.log
│   │   ├── pg.log.2026-04-02.0.log
│   │   └── ...
│   ├── 2/               # 서버 2
│   ├── 3/               # 서버 3
│   └── ...
└── README.md
```

## 요구사항

- Python 3.10 이상 (표준 라이브러리만 사용, 별도 패키지 설치 불필요)

## 사용 방법

### 1. 로그 파일 배치

`pg/` 디렉토리 하위에 서버별 폴더를 만들고 로그 파일을 넣습니다.

```
pg/
├── 1/
│   ├── pg.log.2026-04-01.0.log
│   └── pg.log.2026-04-01.1.log
├── 2/
│   └── pg.log.2026-04-01.0.log
└── ...
```

로그 파일명은 `pg.log.YYYY-MM-DD.N.log` 형식을 따라야 합니다.

### 2. 스크립트 실행

```bash
python analyze_errors.py
```

### 3. 실행 결과

실행하면 콘솔에 서버별 분석 진행 상황이 출력됩니다.

```
총 6개 서버 폴더를 분석합니다.

============================================================
  서버 1 분석 시작
============================================================
  20개 로그 파일 발견
    pg.log.2026-04-01.0.log: 작업 150, 실패 3, Gpki 1
    pg.log.2026-04-01.1.log: 작업 120, 실패 2, Gpki 0
    ...
```

## 출력 파일

스크립트 실행 후 두 종류의 CSV 파일이 자동 생성됩니다.

### 서버별 리포트

각 서버 폴더(`pg/1/`, `pg/2/`, ...) 안에 생성됩니다.

```
pg/1/error_report_server1_20260422_153000.csv
pg/2/error_report_server2_20260422_153000.csv
```

### 날짜별 통합 리포트

`pg/` 디렉토리에 전체 서버를 날짜별로 합산한 리포트가 생성됩니다.

```
pg/error_report_by_date_20260422_153000.csv
```

### CSV 컬럼 구성

| 컬럼            | 설명 |
|---------------|------|
| 파일명 (또는 날짜)   | 로그 파일명 또는 날짜 |
| 전체작업          | Finished Job 총 건수 |
| 성공            | SUCCESS 건수 |
| 전체 실패         | FAILURE 총 건수 |
| 변환 실패         | 실패 - TSA 실패 (순수 변환 실패 건수) |
| TSA 실패        | GPKI 관련 실패 건수 |
| 전체 실패율(%)     | 실패 / 전체작업 |
| 변환 실패율(%)     | 변환 실패 / 전체작업 |
| TSA 실패율(%)    | TSA 실패 / 전체작업 |
| 에러코드별 컬럼      | 각 에러코드의 발생 건수 |
| 실패합계          | 에러코드 건수 합계 |
| [TSA] 메시지별 컬럼 | GPKI 에러 메시지별 발생 건수 |

## 에러코드 사전

`error_codes.py`에 에러코드와 설명이 매핑되어 있습니다. 새로운 에러코드가 추가될 경우 이 파일에 항목을 추가하면 리포트에 설명이 함께 출력됩니다.

```python
ERROR_CODES: dict[str, str] = {
    "BEP000000": "서비스 구동에 실패했습니다",
    "BEP010000": "서비스 초기화에 실패했습니다",
    ...
}
```
